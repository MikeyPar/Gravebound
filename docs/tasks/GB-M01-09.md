# GB-M01-09 — Repeatable projectile-density benchmark

## Task contract

**Feature IDs:** `GB-M01-09`, `TECH-070`, `COM-009`, roadmap package 09
**Authorities:** `Gravebound_Production_GDD_v1_Canonical.md`, `Gravebound_Content_Production_Spec_v1.md`, `Gravebound_Development_Roadmap_v1.md`, and the M01 completion plan
**Dependencies:** Bell Proctor scheduler/primitives, hostile readability priority contract, fixed 30 Hz simulation
**Out of scope:** tuning combat values, altering client encounter ownership, or claiming target-PC acceptance from a simulation-only probe.

## Required behavior

- Construct exactly 800 deterministic hostile projectiles and 40 deterministic enemies from a fixed seed.
- Preserve all hostile telegraphs in full and reduced effect modes. Reduced effects may cull priority 5, then priority 4, only.
- Produce stable state hashes for a 60-second, 1,800-tick stress replay.
- Produce one identical deterministic 60-second Bell Proctor scheduler replay.
- Complete exactly 20 fixed-input Bell Proctor runs with a single shared result hash and a real terminal defeated state.
- Compile performance evidence containing build ID, bundle ID, fixture seed, duration, peak counts, effect mode, frame sample provenance, FPS, p95/p99, target hardware, memory samples/assessment, telegraph retention, acceptance, and a canonical raw-report hash.
- Fail closed on missing identity/samples, zero frame durations, unordered memory samples, invalid effect culling, unverified target class, simulation-only timing, insufficient memory duration, memory over budget, monotonic growth, or missed frame/count/telegraph thresholds.

## Acceptance criteria

1. Fixture count and 60-second golden-hash tests pass.
2. The 60-second boss replay and 20 complete runs produce their checked-in golden hashes.
3. Nearest-rank p95/p99, FPS, report hash, memory duration/budget/trend, sample provenance, target attestation, and reduced-effect policy have direct boundary tests.
4. Focused formatting, tests, and all-target warnings-denied Clippy pass for the owned module.
5. Final ticket promotion remains pending until both full- and reduced-effects rendered runs are captured on verified target-class Windows hardware at 1920x1080 and a real 30-minute resident-memory loop is measured.

## Current implementation state

The renderer-independent deterministic fixture, Bell reliability fixture, evidence schema/compiler, and fail-closed acceptance arithmetic are implemented. Hardware render capture and the 30-minute memory soak remain intentionally unclaimed.
