# GB-M02-03 — Client prediction, reconciliation, and network presentation

## Objective

Make the native client responsive and visually stable over the authoritative M02 session while preserving the server as final for every gameplay outcome.

## Authorities

1. `Gravebound_Production_GDD_v1_Canonical.md`: `SIM-004`, `SIM-010`–`SIM-012`, `TECH-011`–`TECH-014`, and the combat-readability priority contract.
2. `Gravebound_Content_Production_Spec_v1.md`: `CONT-FP-001`–`CONT-FP-008`; predicted movement and projectile presentation consume compiled immutable `fp.1.0.0` definitions and never restate gameplay values.
3. `Gravebound_Development_Roadmap_v1.md`: `GB-M02-03` and the M02 impairment/playability exit conditions.
4. Repository `ADR-020-client-prediction-and-network-presentation.md`.

## Required implementation

- Assemble multi-chunk protocol snapshots only when every unique chunk arrives with identical sequence/tick/state-version/ack metadata. Reject duplicates, inconsistent chunks, entity-ID collisions, and stale completed sequences.
- Predict only local movement at 30 Hz using `sim_core::PlayerMovementState`. Store a bounded sequence-ordered input history.
- On an authoritative player snapshot, discard acknowledged inputs, rebuild from authoritative position and velocity, replay remaining inputs in order, and commit atomically.
- Apply exact `TECH-014` presentation policy to the resulting prediction error:
  - `<0.10 tiles`: blend over `100 ms`.
  - `0.10–0.35 tiles`: blend over `60 ms` and emit a debug metric.
  - `>0.35 tiles`: snap, set a network warning, and emit anomaly telemetry data.
- Accept authoritative alive/dead state immediately. The client cannot predict death or continue movement prediction after authoritative death.
- Interpolate remote entities at a `100 ms`/three-tick delay using stable integer fixed-point samples. Hold outside the known sample interval; do not invent gameplay extrapolation.
- Start locally initiated projectile presentation immediately from compiled content/aim data, tagged by input sequence and within-attack projectile ordinal. Reconcile it to authoritative friendly-projectile snapshots through that stable pair. Present hostile and remote projectiles from authoritative velocity/sample facts.
- Advance the protocol minor because snapshot velocity and source sequence are required wire fields. No compatibility claim is permitted without an adapter.
- Provide a renderer-independent native-client runtime consumed by Bevy presentation code; it owns no sockets, server state, hit detection, health, inventory, reward, eligibility, or death finality.

## Bounds and diagnostics

- Maximum pending local movement inputs: `256`.
- Maximum incomplete snapshot sequences: `4`.
- Maximum interpolation samples/entity: `4`.
- Unconfirmed local projectile presentation expires after `250 ms`.
- Correction records contain distance, class, blend duration, and a closed diagnostic signal (`None`, `DebugMetric`, or `NetworkWarningAndAnomaly`) without account or auth data.

## Acceptance

- Exact threshold tests cover `0`, just below `0.10`, exactly `0.10`, exactly `0.35`, and just above `0.35` tiles.
- Reconciliation with unacknowledged input produces the same state as authoritative reset plus deterministic replay.
- Stale/duplicate/inconsistent chunks cannot alter completed presentation state.
- Remote interpolation is exact at endpoints/midpoints and never extrapolates beyond known state.
- Predicted projectile presentation cannot create hits, damage, death, rewards, or item grants and converges to the authoritative entity/source sequence.
- Focused tests, warnings-denied Clippy, content validation, deterministic traces, and full workspace CI pass before README advances.
