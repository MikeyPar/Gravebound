# GB-M03-11 — PostgreSQL persistence foundation

**Status:** Complete — PASS recorded in `docs/milestones/GB-M03-11-audit.md`.

## Objective

Provide the reusable PostgreSQL connection, migration, transaction, readiness, and disposable integration-test foundation required by M03 without moving product rules into SQL or creating speculative later-domain tables.

## Three design authorities

1. `Gravebound_Production_GDD_v1_Canonical.md`: `TECH-001` through `TECH-006`, `TECH-020`, `TECH-021`, `TECH-030`, and `TECH-060`.
2. `Gravebound_Content_Production_Spec_v1.md`: stable IDs, deterministic/idempotent retry rules, explicit provenance, and stage/promotion boundaries.
3. `Gravebound_Development_Roadmap_v1.md`: `GB-M03-11`, M03 architecture decisions, persistence exit evidence, and the approved ordering in `SPEC-CONFLICT-005`.

## Delivered scope

- Workspace `persistence` crate with redacted PostgreSQL configuration, asynchronous pool, embedded forward migrations, exact readiness report, typed errors, and serializable transaction wrapper.
- Pinned SQLx `0.9.0`, Rust 1.95 compatibility, and PostgreSQL `17.10-alpine3.23` recorded in accepted `ADR-029`.
- Isolated Docker Compose route or explicit disposable `TEST_DATABASE_URL`, protected by an explicit destructive-test opt-in and database-name allowlist.
- Mandatory hosted PostgreSQL gate with no SQLite fallback or silent skip.
- Initial wipeable identity migration limited to the state already approved by `GB-M03-01`; domain-specific migrations remain owned by later M03 packages.

## Acceptance

- Migration replay, schema history/readiness, exact table set, constraints, transaction rollback, ownership, cascade, and typed aggregate round-trip pass against real PostgreSQL.
- Missing local prerequisites fail with actionable messages and nonzero status.
- Strict format, warnings-denied Clippy, workspace tests, deterministic/content gates, and optimized native builds remain green.
- Credentials are environment-only and redacted from `Debug`, `Display`, runtime diagnostics, and committed files.
