**Feature IDs:** `GB-M01-06B`, `DTH-020` presentation subset, `PRD-123`, `QA-100`, `CONT-FP-008`, `CONT-FP-010`

## Required behavior

- Present local hero/class/level/lifetime/final-deed availability before lethal details; unavailable prototype fields must say `N/A`, never be fabricated.
- Bind killer, attack, final damage, damage type, source position, and last five damage events from the immutable authoritative death commit.
- State network mode and the local nonpersistent boundary explicitly.
- Present Lost, Preserved, and Created in order. In M01, Lost is the exact cleanup census; Preserved and Created are `NONE` because accounts, memorials, and Echoes do not exist.
- Expose keyboard-operable primary Run Again; keep the frozen run noninteractive until it succeeds.
- On actual Bell Proctor clear, show clear time, damage taken, potion uses, lethal cause if any, and current/best time.
- Completion-summary Run Again performs the same 06A cleanup/reconstruction transaction; Escape keeps the cleared empty arena; pause exposes the same Run Again action.
- Ask blind testers to name the killer before exposing optional detailed trace inspection.

## Acceptance criteria

1. Cause presentation derives from the committed death snapshot, including run-qualified killer identity and stable pattern ID.
2. Recent timeline order and five-event truncation are deterministic.
3. Run Again remains the primary keyboard action and returns control under three seconds.
4. Actual boss-clear summary, Escape behavior, pause action, and local best-time preservation are proven against the real encounter state.
5. Strict local gates and directly inspected death and victory evidence pass.
