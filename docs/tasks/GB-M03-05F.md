## Delivery contract

- Publish a bounded, versioned Bell checkpoint DTO containing the counter and optional exact pending-repeat projectile snapshot. Canonical bytes and BLAKE3 bind schema, content revision, account, character, instance lineage, aggregate versions, and checkpoint tick.
- Exhaustively validate checkpoint finiteness, ranges, IDs, projectile count/behavior/provenance, delay, damage, digest, lineage, content, and monotonic versions. Corrupt or stale state fails closed.
- Persist dirty live Bell state only on the authored 30-second danger cadence and lifecycle boundaries. Ordinary shots never write PostgreSQL.
- One dormant/test-only Core live aggregate owns compiled combat and consumables by account, character, and instance lineage. Same-instance reconnect rebinds it; dangerous-room handoff preserves Bell state and remaining delay.
- Process resume may reconstruct from the latest valid danger checkpoint. It never changes item security and never replaces the account restore point after an unrecoverable instance crash.
- A committed Danger-to-Safe extraction/Recall transfer resets Bell and deletes its checkpoint only after the durable transaction commits. Failure leaves live state untouched; replay converges idempotently. Dangerous room-to-room transfer does not reset.
- Provide a reusable transaction participant for future death and retirement owners: snapshot ordered active Bargains for memorial/telemetry, delete active rows, reset/delete Bell state, advance aggregate version, and emit one outbox result. Preserve immutable offer/decision history.
- Add exact offered/declined and lifecycle outbox events needed by this boundary. Full telemetry export/redaction remains with the roadmap telemetry package.
- Prove cross-account/character/lineage rejection, stale/corrupt checkpoints, concurrent/replayed writes, disconnect on Bell releases four/five, pending repeat at `+1`/`+8`, room handoff, process restart, safe-transfer ordering, and response-loss replay.
- Prove the real Core identity endpoint remains stage-disabled for combat admission and creates no normal-route scheduler.
