## Objective

Implement the one authoritative transaction that freezes a character at lethal health, destroys every at-risk holding in canonical order, preserves safe state, invokes the mandatory Echo projector, and returns one replayable terminal result.

## Transaction and destruction invariants

- A server-owned terminal arbiter accepts a lethal transition only for the authenticated selected living character in a permadeath-enabled active lineage. It journals one UUIDv7 `death_id` before the SQL attempt and binds the final receipt to `(character_id, "permadeath-v1")`.
- Lock in the repository's canonical order and revalidate life state, lineage, content revision, expected aggregate versions, health zero, terminal journal, at-risk custody, and projector readiness inside the transaction.
- Danger admission is valid only after the `06A` entry component has atomically changed every equipped/Belt unit to `AtRiskEquipped` without changing its location and captured the exact restore snapshot. A safe Equipped/Belt row in danger is corrupt authority and aborts death; it is never silently destroyed or repaired by the terminal transaction.
- Destroy every character-owned item whose security is `AtRiskEquipped` or `AtRiskPending`, including Equipped, both Belt slots' durable units, RunBackpack, PersonalGround, and every run-material-pouch stack. Unknown or internally inconsistent at-risk custody aborts the entire transaction.
- Canonical destruction order is: Equipment `Weapon, Relic, Armor, Charm`; Belt `(slot index, unsigned unit UID)`; RunBackpack index; PersonalGround `(instance_id, pickup_id, item_uid)`; run pouch material ID in UTF-8 byte order. Store contiguous ordinal entries keyed by `(death_id, ordinal)`.
- Each item row transitions to `Destroyed(permadeath)`, advances its item version once, and appends one immutable item-ledger event. Rows and provenance are retained. Run-material stacks receive equivalent immutable destruction entries and become zero only inside the same commit.
- CharacterSafe and Vault items, currency, cosmetics, codex, recipes, mastery, settings, and unrelated account/character state remain unchanged. Their IDs/counts are captured only in the preserved summary projection where authorized.
- The same transaction marks the character dead, advances character/inventory/life versions, writes trace/death/summary/memorial/receipt/audit/outbox rows, clears active Bargains through the existing life-end participant, resets/deletes Bell state, and invokes `GB-M03-13` before commit.
- Exact retry returns the stored terminal result even after response loss or restart. Changed payload, foreign binding, stale nonreplay, nonzero health, later mutation, or a losing Recall/extraction request receives a typed nonmutating result.
- A committed death wins any same-tick terminal conflict. If no terminal transaction committed before a verified server crash, ordinary danger-entry restoration applies and no death is synthesized.
