# GB-M03-05D - Deterministic Veil Bargain offer and shrine choice

**Status:** Complete - see [`GB-M03-05D-audit.md`](../milestones/GB-M03-05D-audit.md).

Deliver the temporary Core level-5 Sepulcher Knight milestone, exact deterministic three-choice Bargain offer, durable select/refuse state, authoritative shrine projection, and accessible choice UI without enabling Bargain mechanics, purge, death cleanup, or the normal route.

## Approved implementation contract

- The Core trigger is only the first `miniboss.sepulcher_knight` clear in `layout.core_private_life_01` at level 5 or above. It is disabled in Slice and later bundles.
- A shared `character_oath_bargain_state` aggregate owns positive version, earned slots `0..=3`, and active Bargains in acquisition order. It is the real provider for the existing `OathBargainRestoreV1` seam.
- Qualifying milestone resolution, earned-slot grant, immutable offer creation, and any exact 10-Ash fallback share the progression transaction. A second best-effort transaction is not legal.
- Offer identity is the source reward-event idempotency key. Replay returns the exact prior disposition; concurrent duplicates create one slot/offer or fallback award.
- Candidate planning uses the exact `CONT-014` BLAKE3 preimage, raw 16-byte reward/character UUIDs, length-prefixed immutable content version and UTF-8 candidate ID, unsigned score ordering, and candidate-ID tie break.
- Candidates come only from enabled stage content, exclude active Bargains, satisfy `BRG-004`, and remain within resolved-stat caps. The exact Core catalog normally yields Bell Debt, Cinder Hunger, and Lantern Ash in score order.
- Persist candidate IDs, scores, and order immutably. Emergency one/two-candidate offers show disabled `UNAVAILABLE` cells; zero candidates resolve unavailable, retain the earned slot, and grant exactly 10 Ash once.
- An open offer transitions exactly once to `Selected(offered_id)` or `Refused`. Selection revalidates authority/content/legality and persists the active Bargain before success returns. Refusal has no penalty and leaves the earned slot unfilled. Escape closes UI and is never refusal.
- Mutation replay is checked before current state. Identical retries return the byte-identical stored result; changed material conflicts. Accepted selection and its life event commit atomically.
- The UI shows three ordered cells, exact boon/curse copy, explicit selection confirmation, refusal, unavailable cells, in-flight lockout, and authoritative before/after health, damage, cooldown, movement, healing, plus attack-rate or belt-slot axes when affected. It must support pointer, keyboard, controller, and non-color labels.
