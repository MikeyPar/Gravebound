# GB-M02-08 - Instance lifecycle, scheduler diagnostics, and M02 automated gate

## Objective

Host the existing authoritative logical sessions under explicit bounded arena-instance ownership, measure production-shaped scheduler work, retire all resolved state, and prove the complete automated M02 gate with sixteen protocol-driven bots for two simulated hours.

1. `Gravebound_Production_GDD_v1_Canonical.md`: `SIM-003`-`SIM-014`, `TECH-001`-`TECH-015`, `TECH-060`, `TECH-070`, and `TECH-080`.
2. `Gravebound_Content_Production_Spec_v1.md`: immutable `fp.1.0.0` remains the only M02 gameplay bundle. M03 micro-realm and dungeon state machines are not enabled early.
3. `Gravebound_Development_Roadmap_v1.md`: `GB-M02-08` requires realm/arena instance lifecycle and diagnostics; the M02 exit gate requires four humans, sixteen bots for two hours, the 100/20/1 impairment profile, p95/p99 tick limits, and all malicious/mutation tests.
4. Repository `ADR-025-bounded-instance-host-and-soak-evidence.md`.

## Required implementation

- Add a `server_app` instance module. An `InstanceScheduler` owns every hosted instance; each hosted instance owns exactly one `SessionDirectory`; every logical session owner maps to at most one instance.
- Model closed phases `Allocating -> Active -> Draining -> Closed`. First accepted admission activates an instance. Draining rejects new admission. Closed instances own no sessions or transports.
- Pin each instance to immutable content bundle `fp.1.0.0` and one instance kind. M02 executes `CombatArena`; `Realm` is a typed routing identity only until M03 supplies approved world content.
- Deterministically admit a new owner to the oldest active instance with capacity, tie lowest instance ID, or allocate a new instance. Reconnect/control/gameplay routing uses the owner index and never scans into a different authority aggregate.
- Bound M02 arena capacity at sixteen concurrent logical sessions, the exact automated exit population. Bounds are infrastructure/test capacity, not future realm balance values.
- Advance every simulation-active session exactly once per scheduler frame. A frame fails closed if a session stalls, advances by more than one tick, produces invalid output, or exhausts diagnostics.
- Retire Dead, Recalled, and Closed sessions explicitly, remove their owner index entries, and close an empty instance. Resolved state cannot accumulate across successor cycles.
- Graceful scheduler shutdown stops admission, enters Draining, uses the existing per-directory shutdown transaction, closes transports without death, clears directories, closes instances, and leaves zero membership/index entries.
- Record bounded scheduler diagnostics: frame count, session steps, admissions, reconnects, retirements, instance allocations/closures, invalid-state/stall counts, a fixed-memory rolling tick-time window, nearest-rank p95/p99, mean work, and CPU headroom against the 33,333 microsecond 30 Hz budget.
- Preserve at least 30% measured mean CPU headroom; require p95 at most 20,000 microseconds and p99 at most 30,000 microseconds.
- Run sixteen `JourneyBot` policies for exactly `216,000` scheduler frames (two hours at 30 Hz). Traffic crosses canonical protocol encode/decode boundaries and exercises live combat, pickup, Recall/death finality, resolved-session retirement, and successor re-admission rather than idle sockets.
- Sample actual process resident memory after warmup at deterministic simulated-time checkpoints. Reuse the existing performance compiler's `8 MiB` monotonic-growth floor so allocator page noise is not mislabeled as a leak; any strictly monotonic growth at or above that floor fails. Any growth in bounded logical instance/session/index counts fails independently with no tolerance.

## Bounds and ordering

- M02 arena capacity: `16` logical sessions.
- Soak duration: `2 hours = 216,000` scheduler frames at 30 Hz.
- Tick samples retained: a fixed rolling window of exactly the full M02 soak duration (`216,000` frames). The server can run indefinitely without allocating or exhausting diagnostics; the explicit soak fills the window exactly once.
- Tick percentiles use nearest rank: `ceil(N * percentile / 100)`, one-indexed.
- Mean headroom basis points: `10,000 - ceil(mean_tick_micros * 10,000 / 33,333)`; pass requires at least `3,000`.
- Membership/index insertion, reconnect routing, retirement, and instance close are transactional: validation and all checked arithmetic complete before externally visible ownership changes.
- Scheduler simulation and terminal precedence remain those of `ManagedSession`; the instance layer cannot reinterpret death, Recall, inventory, or combat.
