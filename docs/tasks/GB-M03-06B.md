## Deterministic simulation contract

- `lifetime_ticks` advances at authoritative 30 Hz only while a living character is controllable in Hall or danger. It includes the vulnerable 90-tick LinkLost window and excludes character select, loading, and offline time. Persist `lifetime_ms = floor(lifetime_ticks * 1000 / 30)` with checked integer arithmetic.
- `permadeath_combat_ticks` advances while living after committed danger entry through death, extraction, or Recall, including staging and vulnerable LinkLost. Crash restoration rolls it back to the value captured at entry. Echo time eligibility is exactly `>= 18_000` ticks.
- Persist reward-qualified completion IDs idempotently. Echo deed eligibility requires at least one dungeon-boss completion ID or at least two distinct major-realm-event IDs during that life; rejected, practice, duplicate, dead, Recalled, or ineligible rewards do not count.
- Add stable deed IDs `deed.core.sepulcher_knight_defeated` and `deed.core.sir_caldus_defeated`. Select the latest by `(achieved_tick, deed_id)`; if no qualified deed exists, use `deed.none` with exact en-US copy `No final deed recorded.` Localization is presentation only; the death row stores the ID.
- Reuse and generalize the existing local ten-second trace behavior: retain ordered authoritative damage entries inside the final 300-tick window, allow stable same-tick entries, and order by `(tick, event_ordinal)`. The lethal entry is always present and uniquely flagged.
- Each entry captures source/killer identity, pattern/attack, raw/final damage, type, pre/post health, fixed-point source position, statuses, and relevant network/Recall state from the authoritative tick. Unknown/nonfinite data, health arithmetic disagreement, tick regression, or inconsistent lethality rejects terminal staging.
- Deterministic cause selection uses the accepted health-zero transition. Equal-tick inputs follow the simulation's canonical event order; neither client arrival order nor localization can choose the cause. The summary's last-five timeline is the final five ordered trace entries, oldest to newest.
- Clock, deed, and trace state participate in danger checkpoint/process-resume records as investigation state only. An unrecoverable crash with no committed terminal transaction restores the danger-entry values and revokes post-entry unsecured gains.
