# GB-M02-02 — Authoritative gameplay session

## Objective

Route one authenticated player's complete combat-test state through the authoritative server at 30 Hz while retaining `sim_core` as the only owner of movement, attack, cooldown, projectile, collision, health, death, eligibility, and pickup rules.

## Authorities

1. `Gravebound_Production_GDD_v1_Canonical.md`: `SIM-004`–`SIM-006`, `SIM-010`–`SIM-014`, `TECH-011`–`TECH-014`, `DTH-001`, `LOOT-002`, and `LOOT-010`.
2. `Gravebound_Content_Production_Spec_v1.md`: `CONT-FP-001`–`CONT-FP-008`; the network combat test consumes the validated immutable `fp.1.0.0` bundle and may not restate its numeric gameplay values.
3. `Gravebound_Development_Roadmap_v1.md`: `GB-M02-02` and the M02 objective/exit gate.
4. Repository `ADR-019-authoritative-session-boundary.md`.

## Required implementation

- Add a renderer- and transport-independent `sim_core` aggregate that transactionally composes the validated arena, player movement/combat/vitals, normal-enemy wave, projectiles, collision, inventory, death finality, reward eligibility, and personal pickup state.
- Construct the aggregate from definitions compiled by `sim_content`; production server code must not call prototype convenience constructors or copy content numbers.
- Accept compact fixed-point movement/aim and sequenced primary/ability intent. Only the server clock advances simulation; client tick values are diagnostic input metadata.
- Keep latest-state input coalescing separate from reliable ordered actions. A server tick consumes one coherent input/action state exactly once.
- Emit 15 Hz typed snapshots for the player, enemies, friendly/hostile projectiles, and available personal pickups. Health/death and phase state are authoritative.
- Add an idempotent reliable pickup mutation with a 128-bit mutation ID, typed result code, server-owned eligibility check, exact sim-owned reach/capacity rules, and monotonic state version.
- Once death commits, reject further gameplay and mutation commands, clear run-owned inventory/projectiles, and report death only from authoritative health state. Durable character-death persistence remains M03.
- Run the real codec and QUIC transport in integration coverage; no test-only bypass may author an accepted gameplay result.

## Security and fail-closed bounds

- Reject session mismatch, invalid fixed-point vectors, zero/nonmonotonic reliable action sequence, mutation-ID zero, invalid pickup identity, wrong message/channel, and state-version overflow.
- Older datagram input is ignored as superseded, never re-applied. Detailed malicious movement/fire/pickup suites remain `GB-M02-06`, but M02-02 must establish the closed authority seam they attack.
- Authentication tickets and opaque session credentials never enter gameplay logs or snapshots.

## Explicitly deferred

- Client prediction/interpolation/reconciliation: `GB-M02-03`.
- Disconnect, LinkLost, Recall completion, duplicate-session handoff, and shutdown: `GB-M02-04`.
- Impairment and comprehensive malicious-input harnesses: `GB-M02-05`/`GB-M02-06`.
- Full journey bot and multi-instance lifecycle: `GB-M02-07`/`GB-M02-08`.
- Accounts, durable item/death transactions, and PostgreSQL: M03.

## Acceptance

- A deterministic scripted session proves movement, primary and both ability sequences, cooldown ownership, friendly/hostile projectiles, collision damage, enemy/player health, enemy/player death, eligibility, and one idempotent pickup.
- Client-supplied position, health, hit, reward, or item-grant outcomes have no accepted protocol representation.
- Duplicate pickup returns the original result without a second mutation; ineligible/dead/out-of-range requests cannot mutate state.
- Snapshot cadence is exactly every two 30 Hz ticks and chunks remain within M02-01 bounds.
- Focused tests, warnings-denied Clippy, content validation, deterministic traces, and full workspace CI pass before the audit advances README.
