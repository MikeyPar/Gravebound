# GB-M02-04 — Connection lifecycle and reconnect

## Objective

Own the complete logical-session lifecycle around the authoritative combat session so transport loss cannot duplicate authority, erase a valid death, or become a faster escape than Emergency Recall.

## Authorities

1. `Gravebound_Production_GDD_v1_Canonical.md`: `TECH-010`–`TECH-015`, `DTH-010`, and the server-authority contract.
2. `Gravebound_Content_Production_Spec_v1.md`: immutable `fp.1.0.0` remains the only gameplay bundle; reconnect restores server-owned state and a resolved disconnect cannot receive rewards.
3. `Gravebound_Development_Roadmap_v1.md`: `GB-M02-04` join, leave, timeout, LinkLost, reconnect, duplicate-session handling, and clean shutdown.
4. Repository `ADR-021-logical-session-and-transport-handoff.md`.

## Required implementation

- Separate authenticated logical-session identity from replaceable QUIC transport identity. One owner has at most one live logical session in the M02 directory.
- Carry join, reconnect, and leave as bounded reliable Control-channel requests with typed results. Advance the exact-match protocol minor for the wire change.
- Join creates one authoritative session. Repeated join by the same authenticated owner performs the same atomic transport handoff as reconnect; it never creates a second authority aggregate.
- Transport loss or accepted leave enters `LinkLost` at the current authoritative tick, neutralizes movement/held attack input, keeps the character vulnerable, and sets the inclusive resolution boundary to `lost_tick + 90`.
- Continue stepping the existing authoritative session during LinkLost. On every tick, commit authoritative simulation and death first; only a still-living character at or beyond the deadline commits automatic Recall.
- Reconnect before resolution rebinds the same authoritative session, state version, inventories, health, and entity identities. It returns an immediate time-sync sample.
- Reconnect after Recall routes to Lantern Halls. Reconnect after death reports final death. Neither outcome reconstructs combat state on the client.
- A duplicate connection invalidates the former transport only as part of a successfully validated, atomic handoff. All later input/control/reliable messages from the former transport fail closed.
- Client connection presentation may show Connected, LinkLost, Reconnecting, Lantern Halls, Death Final, or Server Shutting Down, but a local countdown cannot decide Recall or death.
- Graceful shutdown stops admission, emits `ServerShuttingDown`, closes every logical session without death, and permits deterministic teardown. An unclean server/instance loss is not modeled as character death.

## Bounds and ordering

- LinkLost duration: exactly `3 seconds = 90` authoritative ticks at 30 Hz.
- Logical session, owner, transport, request, and tick identities are nonzero and checked for exhaustion.
- Control request sequences are strictly increasing per transport.
- Lifecycle clock cannot move backward.
- Session-result IDs use the existing bounded 64-byte wire identifier.
- The old transport remains current if owner validation, session validation, sequence validation, or handoff construction fails.
