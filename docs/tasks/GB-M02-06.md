# GB-M02-06 — Malicious input and mutation rejection

## Objective

Prove that encoded hostile or replayed client traffic cannot move, fire, hit, grant, duplicate, resurrect, or stall the authoritative combat session outside the same bounded intent rules used by an ordinary client.

## Authorities

1. `Gravebound_Production_GDD_v1_Canonical.md`: `SIM-004`, `SIM-010`–`SIM-012`, `TECH-011`–`TECH-015`, and `TECH-120`–`TECH-125`.
2. `Gravebound_Content_Production_Spec_v1.md`: immutable `fp.1.0.0` supplies all speed, cooldown, collision, reach, reward, and inventory facts; the abuse layer cannot invent alternate gameplay values.
3. `Gravebound_Development_Roadmap_v1.md`: `GB-M02-06` teleport, speed, fire-rate, forged-hit, duplicate-pickup, stale/replayed-input, and mutation tests.
4. Repository `ADR-023-fail-closed-gameplay-ingress.md`.

## Required implementation

- Keep the protocol intent-only. No client wire variant may carry position, velocity, collision result, hit, damage, health, death, eligibility, drop, reward, or item-grant authority.
- Decode and validate every adversarial fixture through the canonical codec before managed-session ingress.
- Coalesce bounded Input datagrams by strictly newer input sequence. Duplicate/stale datagrams do not mutate state and are classified as expected network disorder, not automatic cheating.
- Reject a held-primary source sequence older than the highest previously observed press. Equal sequence remains legal for held fire; cadence and projectile creation remain entirely server cooldown/state outcomes.
- Ability sequence fields on Input datagrams must remain zero because ability presses use reliable Action messages.
- Convert stale/replayed reliable Action messages to typed `StaleSequence` results. Accept at most one pending press per ability per authoritative tick; extra presses return `RateLimited` without overwriting the accepted press.
- Bind every mutation ID to its original complete request. Exact retry returns the cached result; reusing the ID with a different pickup or placement returns typed `IdempotencyConflict` and never executes either new payload.
- Bound accepted new mutation attempts and the idempotency cache. Duplicate pickup under a new mutation ID returns `AlreadyResolved`; out-of-range, dead, ineligible, unknown, and capacity paths remain typed and nonmutating.
- Maintain bounded privacy-safe ingress diagnostics. Ordinary stale datagrams are counted separately from suspicious primary regression, action flood/replay, idempotency conflict, and rejected mutation behavior. Diagnostics never ban or mutate gameplay.
- Prove teleport, speed, and forged-hit attempts cannot be represented as valid protocol messages; header/payload substitution or unknown variants fail codec validation.

## Technical bounds

- Maximum accepted new mutations per authoritative tick: `8`.
- Maximum cached mutation identities per logical combat session: `1024`.
- Maximum retained anomaly records: `64`.
- Diagnostics use saturating counters and a reviewed evidence score only; no automatic permanent enforcement exists in M02.

## Explicitly deferred

- Edge DDoS/connection quotas, platform anti-tamper signals, staff review/enforcement, and durable security audit storage: M06.
- Replayed craft and purchase callbacks require their later protocol/persistence domains.
- Full journey bot: `GB-M02-07`; multi-instance ownership: `GB-M02-08`.

## Acceptance

- Encoded tests cover out-of-range vector, illegal ability-in-input, regressing primary sequence, input replay/reorder, input flood/coalescing, action replay, per-ability flood, cooldown/fire cadence, forged wire kind/payload, duplicate pickup with same/different mutation IDs, idempotency conflict, cache/rate bounds, out-of-range, ineligible, and dead mutation.
- Every rejection is typed where a semantically valid message reaches a response channel; malformed/forged frames fail codec validation before gameplay ingress.
- Rejected cases leave authoritative state version, health, position, inventory, pickup collection, projectile census, and accepted sequencing unchanged as applicable.
- Identical abuse scripts produce identical results, diagnostics, and authoritative terminal state.
- Networking and full workspace gates pass before README advances.
