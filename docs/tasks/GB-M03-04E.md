## Implementation contract

- The aggregate exposes exactly four typed equipment slots and eight RunBackpack indices. Only equipment with a matching authored legal slot may enter `Equipped`.
- A preview is a pure, content-bound projection over the authoritative inventory version. It names the incoming item, current item, exact replacement destination, resolved behavior/stat differences, and whether confirmation is legal.
- `RunBackpack -> Equipped` moves the replaced item into the exact source index vacated by the incoming item. A full backpack therefore remains legal.
- `PersonalGround -> Equipped` moves the replaced item to the lowest empty RunBackpack index. If none exists, preview and commit reject without moving either item.
- Confirmation binds account, character, command identity, expected inventory version, source identity/location, target slot, exact content revision, and preview hash. Stale, altered, duplicate, cross-owner, or mismatched commands fail closed; identical retry returns the stored result.
- One accepted swap atomically updates both item rows, appends immutable ledger entries, advances each changed item version once, advances the inventory version once, and stores the replay result. No partial mutation may be observable.
- Resolved equipment behavior follows the compiled Core definitions and documented item resolution order. A stat rebuild preserves absolute current health on maximum-health increase and clamps to the new maximum with a living floor of one on decrease; rebuilding neither heals nor kills.
- The native inventory opens on `I`/`Tab` without pausing the world, protects the center/lower-middle playfield, blocks combat input only while the pointer/modal owns interaction, and supports pointer, keyboard, and controller-focus metadata.
- The default comparison order is behavior first, then damage/defense changes, then advanced numerical detail. Pending items have a non-color ash treatment and the exact `Lost on death or Emergency Recall` warning.
- Field equip requires select plus explicit confirmation. The confirmation surface names the replaced item's destination before mutation, locks while in flight, and exposes authoritative success/rejection/retry state.
- All 18 Core icons render from the approved 04C source/runtime mapping at 64x64 and minimum supported scale. Evidence covers standard and reduced-effects presentation at 1280x720 and 1920x1080; text never renders below the effective 14px floor.
