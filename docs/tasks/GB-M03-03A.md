# GB-M03-03A — Unpromoted Core world-flow content

**Status:** Complete — PASS recorded in `docs/milestones/GB-M03-03A-audit.md`.

## Objective

Compile the exact Core Lantern Halls and private microrealm geometry, child records, graybox asset references, and localization under an independently hashed, non-promotable development target. This package must make later world-flow implementation possible without activating a player route or inventing dungeon, encounter, item, Oath, death, extraction, or Recall behavior.

## Three design authorities

1. `Gravebound_Production_GDD_v1_Canonical.md`: `PRD-130`, `LOOP-001`, `DTH-001`, `DTH-010`, `DTH-011`, `TECH-020`–`023`, `TECH-060`, `WRLD-001`–`004`, `DNG-001`–`005`, and the safe-Hall requirements.
2. `Gravebound_Content_Production_Spec_v1.md`: `CONT-001`–`003`, `CONT-WORLD-001`, `CONT-ROOM-001`–`008`, `CONT-HUB-001`–`002`, `CONT-HUB-VALID-001`, and `CONT-VALID-001`–`003`.
3. `Gravebound_Development_Roadmap_v1.md`: `GB-M03-03`, the M03 private-life objective/exit gate, and the approved `SPEC-CONFLICT-006` split and route gate.

## Scope

- Strict typed schemas for an unpromoted target, development headers, fixed-point geometry, Hall/world records, semantic child variants, graybox assets, and localized copy.
- One exact hub, one exact world, and ten exact child records with derived IDs, assets, tags, parents, coordinates, and Core availability.
- Independent BLAKE3 pins for record, asset, and localization bytes.
- Exact Hall shell, solids, spawns, five Core stations, prohibited creation capabilities, and radius-aware reachability.
- Exact capacity-one microrealm shell, gate, spawn, safe fork, Bell portal, width-five road, nine candidate anchors, first-eight selection, and disabled larger-world systems.
- Explicit `core_world_flow_integration` gates on player-route and station surfaces.
- Dedicated validation command and adversarial tests.

## Exclusions

Runtime route admission, destination bindings, transfer protocol/persistence, encounter pack activation, Bell rooms/layout/enemies/boss, item/Oath/death/extraction/Recall behavior, public or party admission, `core.1.0.0`, release manifests, packages, and promotion records.

## Acceptance criteria

1. The exact 12-record allowlist and all cross-record asset/localization/parent references compile from strict deny-unknown schemas under pinned file hashes.
2. Validation proves Lantern Halls is safe and navigable and the microrealm is exact, capacity one, deterministic, and free of public/party/larger-world systems.
3. No route, encounter, room, secret, release, package, or promotion binding exists; immutable `fp.1.0.0` bytes remain unchanged.
4. Schema generation, strict content commands, warnings-denied workspace lint, all workspace tests, deterministic headless replay, and optimized Windows workspace build pass.
