## Player-visible outcome

The stationary Chain Sentry warns and activates two centered arena-collision lanes, alternates cardinal and diagonal axes, and damages a given player at most once per cast.

## Exact rules

- FP override: Anchor, health `300`, armor `5`, hurtbox `0.55`, speed `0`, aggro/leash `13` tiles.
- State order: `SpawnTelegraph(900) -> Dormant(700) -> LaneTelegraph -> LaneImpact(350) -> RecoverUntilCycle(4500) -> ToggleOrientation`.
- First axes `[0°,90°]`, next `[45°,135°]`, alternating thereafter. Two centered lanes extend to authored arena collision; width `0.9`.
- First warning `800 ms`, repeated warning `650 ms`, active `350 ms`; physical Pressure damage `22` once/player/cast; threat `12` per lane; memory `lane_or_beam`; counterplay `leave_telegraph`; disposition `expire_at_authored_end`; maximum active instances `2`.
- Resolved values: spawn `27`, dormant `21`, cycle `135`, first warning `24`, repeated warning `20` (ceiling), active `11` ticks.
- The active cast ID is the contact identity. A player ID is accepted once, duplicate contact returns false, and a different player remains eligible. Contact history is deterministic and cleared only with cast lifecycle.
- Exact pattern optional fields matter: projectile speed/radius/lifetime must be absent; lane width/active must be present.

## Ownership and failure behavior

- `sim_core::enemy` owns definition, orientation, cast IDs, telegraph/active/expiry events, per-cast player-contact set, and trace.
- `sim_content` validates exact state/pattern/reference tuples and constructs the immutable definition with shared `DamageType::Physical` and `DamageBand::Pressure`.
- Renderer lane geometry, actual player intersection, damage resolution, health/death, rewards/drops, and persistence are downstream.
- Wrong cast, inactive cast, duplicate/invalid definition, overflow, or content drift fails closed without inventing damage.
