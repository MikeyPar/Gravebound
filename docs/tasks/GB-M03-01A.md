# GB-M03-01A — Immutable FP boundary and unpromoted Core development compiler

## Objective

Create a fail-closed content boundary for Core identity work without changing, copying, promoting, or repackaging any `fp.1.0.0` record. The target resolves only the stable Grave Arbalist class, its four level-1 abilities, and the existing base sprite authorized by `SPEC-CONFLICT-004`.

## Authorities

1. `Gravebound_Production_GDD_v1_Canonical.md`: `CLS-001`, `CLS-002`, `CLS-020`, `TECH-002`, `TECH-004`, and `TECH-100` through `TECH-105`.
2. `Gravebound_Content_Production_Spec_v1.md`: `CONT-000` through `CONT-003`, `CONT-FP-001`, `CONT-VALID-001`, and `CONT-VALID-003`.
3. `Gravebound_Development_Roadmap_v1.md`: M03 objective and package `GB-M03-01`.
4. `docs/spec-conflicts/SPEC-CONFLICT-004-m03-core-identity.md`: approved decisions 1, 3, and 6.

The GDD controls class behavior and typed content ownership, the Content Production Specification controls exact IDs and release semantics, and the roadmap controls delivery order.

## Scope

- Preserve every tracked First Playable content byte and continue validating the exact `fp.1.0.0` package.
- Add a strict, checked-in `core_dev` identity descriptor whose type has no release-stage, bundle, promotion, packaging, or deployment fields.
- Reuse the validated `class.grave_arbalist` record and exact ability IDs:
  - `ability.arbalist.primary_crossbow`
  - `ability.arbalist.grave_mark`
  - `ability.arbalist.slipstep`
  - `ability.arbalist.stillness`
- Resolve only `sprite.class.grave_arbalist` for the locked Core identity preview. It is not an appearance record or entitlement.
- Compile a non-serializable immutable runtime view for identity consumers.
- Reject missing, duplicate, reordered, extra, prototype, item, arena, and reward IDs.
- Reject relabeled source versions, drifted stable class/ability records, and attempted release/promotion metadata.
- Check the JSON Schema contract into `schemas/` and prove it matches the Rust type.

## Explicitly deferred

- The complete Core manifest, `core.1.0.0` release package, `ContentPromotion`, and `CONT-VALID-003` promotion gates.
- Core items, rewards, arenas, rooms, enemies, oaths, Bargains, localization breadth, and production appearance records.
- Identity authority/protocol (`GB-M03-01B`) and native identity presentation (`GB-M03-01C`).
- PostgreSQL, durable persistence, Lantern Halls, world transfer, inventory, and item power.

## Acceptance

- The checked-in Core development descriptor compiles exactly one class, four abilities in canonical class order, and one base sprite from fully validated `fp.1.0.0` source records.
- The compiled type cannot serialize or contain a release manifest, bundle ID, release stage, promotion record, or package output.
- Strict schema decoding rejects invented promotion/release fields.
- Semantic validation rejects any prototype item, arena, or reward leakage before allowlist resolution, and rejects every other allowlist drift.
- Reviewed BLAKE3 byte locks cover all FP domain JSON, the exact release/asset manifests, and FP localization; Git reports no diff in those files.
- Focused tests, warnings-denied Clippy, format checks, and strict existing content validation pass.
- `core.1.0.0` does not exist under manifests, packages, or promotions.
