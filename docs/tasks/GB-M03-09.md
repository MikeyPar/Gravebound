# GB-M03-09 - Versioned privacy-safe telemetry

**Status:** Typed telemetry package plus the first durable PostgreSQL domain-source foundation are implemented locally. Runtime wiring, loot source binding, hosted proof, destination/privacy review, and final operational evidence remain open.

## Objective

Provide versioned onboarding, session, loot, extraction, Emergency Recall, death, successor, and crash telemetry that is sourced only from committed domain outbox records, cannot serialize raw account/network secrets, and remains bounded and optional when offline or disabled.

## Three design authorities

1. `Gravebound_Production_GDD_v1_Canonical.md`: `TECH-005`, `TECH-123`, and `TEL-001`-`005` define the modular boundary, privacy constraints, common envelope, event taxonomy, death detail, KPI uses, and evidence gates.
2. `Gravebound_Content_Production_Spec_v1.md`: the Core stable IDs, content revision, and lifecycle boundaries are the content authority carried by events. Telemetry does not invent or localize gameplay facts.
3. `Gravebound_Development_Roadmap_v1.md`: roadmap `ADR-005` and `GB-M03-09` require a dedicated versioned telemetry/crash/privacy pipeline with bounded batching and disabled/offline behavior.

## Delivered contract

- Dedicated `telemetry` workspace crate with a schema-version-1 `TEL-001` envelope and typed onboarding, session, loot, extraction, Recall, death, successor, and client/server crash events.
- Derived event names, bounded canonical stable IDs/tags, validated death fields, and append-only versioning expectations.
- Fixed-width pseudonymous account correlation with redacted debug output. No field exists for raw account identity, IP/socket address, authentication material, email, platform identity, free-form crash message, stack trace, or arbitrary properties.
- Redacted serialization is available only through the pipeline; callers cannot serialize a bare envelope through the crate API.
- Ingestion accepts only a committed outbox projection. Queue idempotency is keyed by durable outbox ID, and delivery acknowledgement removes only exact accepted IDs.
- Disabled mode retains and exports nothing. Offline mode queues up to the configured bound, never exports while disconnected, never silently evicts, and returns explicit backpressure at capacity. Export batches are capped.
- The PostgreSQL adapter polls the existing committed death, extraction, Recall, and successor outbox families in bounded order, decodes their canonical stored payloads, and maps them to typed events. Death enrichment reads only the immutable committed death/summary/trace graph; the adapter never reads mutable gameplay state.
- Additive migration `0070_m03_telemetry_domain_sources_v1.sql` adds durable logical sessions and typed onboarding, session, and crash outboxes. Account creation, character creation, and first combat are projected only by triggers in their owning gameplay transactions; rollback creates no telemetry fact and absence of an open telemetry session does not reject the gameplay write.
- Every schema-0070 source row binds its origin session. Polling joins that immutable per-row build/content/platform/region/environment/cohort context, so delayed publication or restart cannot relabel an event with a newer live session.
- Typed persistence repositories own session start/disconnect/reconnect/end and redacted client/server crash observations. Exact retries return stored facts, changed payloads fail closed, one open logical session is allowed per account, server-derived durations are canonical, and publication may advance only once.
- Account IDs are converted inside the export adapter with a redacted keyed pseudonymization secret. The existing terminal-family adapter still receives authoritative projection context from its caller; replacing that temporary boundary with durable per-row session binding remains required before runtime enablement.
- The disabled-by-default worker exports privacy-safe documents through an idempotent exporter seam. Failed responses cause no source mutation. Exact accepted IDs advance existing `published_at` markers atomically; unaccepted rows survive process loss and are re-polled after restart.
- [`ADR-039`](../decisions/ADR-039-telemetry-outbox-privacy-and-retention.md) records the privacy, ingestion, retention, and deletion boundary.

## Explicit exclusions

- Gameplay writes or decisions, direct live-handler emission, arbitrary JSON fields, raw crash dumps, raw identity lookup, or a second durable telemetry database.
- M04+ party/commerce/achievement events, dashboards, product KPI decisions, remote collector procurement, and legal/privacy approval.
