# GB-M03-09 - Versioned privacy-safe telemetry

**Status:** Typed telemetry, durable PostgreSQL sources, optional production-root logical-session wiring, typed redacted next-launch native crash collection, immutable schema-71/72 item-ledger binding, schema-73 terminal-family origin binding, and explicit disabled production-root worker ownership are implemented locally. Complete `TEL-003` death inputs, hosted execution, destination/privacy review, enabled adapter/exporter wiring, and final operational evidence remain open.

## Objective

Provide versioned onboarding, session, loot, extraction, Emergency Recall, death, successor, and crash telemetry that is sourced only from committed domain outbox records, cannot serialize raw account/network secrets, and remains bounded and optional when offline or disabled.

## Three design authorities

1. `Gravebound_Production_GDD_v1_Canonical.md`: `TECH-005`, `TECH-123`, and `TEL-001`-`005` define the modular boundary, privacy constraints, common envelope, event taxonomy, death detail, KPI uses, and evidence gates.
2. `Gravebound_Content_Production_Spec_v1.md`: the Core stable IDs, content revision, and lifecycle boundaries are the content authority carried by events. Telemetry does not invent or localize gameplay facts.
3. `Gravebound_Development_Roadmap_v1.md`: roadmap `ADR-005` and `GB-M03-09` require a dedicated versioned telemetry/crash/privacy pipeline with bounded batching and disabled/offline behavior.

## Delivered contract

- Dedicated `telemetry` workspace crate with a schema-version-1 `TEL-001` envelope and typed onboarding, session, loot, extraction, Recall, death, successor, and client/server crash events.
- Derived event names, bounded canonical stable IDs/tags, validated death fields, and append-only versioning expectations.
- Fixed-width pseudonymous account correlation with redacted debug output. No field exists for raw account identity, IP/socket address, authentication material, email, platform identity, free-form crash message, stack trace, or arbitrary properties.
- Redacted serialization is available only through the pipeline; callers cannot serialize a bare envelope through the crate API.
- Ingestion accepts only a committed outbox projection. Queue idempotency is keyed by durable outbox ID, and delivery acknowledgement removes only exact accepted IDs.
- Disabled mode retains and exports nothing. Offline mode queues up to the configured bound, never exports while disconnected, never silently evicts, and returns explicit backpressure at capacity. Export batches are capped.
- The PostgreSQL adapter polls the existing committed death, extraction, Recall, and successor outbox families in bounded order, decodes their canonical stored payloads, and maps them to typed events. Death enrichment reads only the immutable committed death/summary/trace graph; the adapter never reads mutable gameplay state.
- Additive migration `0070_m03_telemetry_domain_sources_v1.sql` adds durable logical sessions and typed onboarding, session, and crash outboxes. Account creation, character creation, and first combat are projected only by triggers in their owning gameplay transactions; rollback creates no telemetry fact and absence of an open telemetry session does not reject the gameplay write.
- Every schema-0070 source row binds its origin session. Polling joins that immutable per-row build/content/platform/region/environment/cohort context, so delayed publication or restart cannot relabel an event with a newer live session.
- Additive migration `0071_m03_loot_telemetry_origin_v1.sql` projects only canonical `item_ledger_events` inside their owning transaction. The immutable sidecar binds each emitted item-created, picked-up, equipped, extracted, or destroyed fact to the one durable session interval covering that ledger transaction plus the item UID, template, reward/starter source, item version, and occurrence time present at commit. A later replacement session cannot relabel an older in-flight mutation. Missing or ambiguous session coverage cleanly skips projection without rejecting the gameplay write or waiting on session shutdown; pre-0071 history is not guessed or backfilled.
- The schema-71 repository polls only the immutable sidecar and its bound session context with a hard 256-row limit. It never reads mutable item state or rewrites item/reward history. Exact accepted event IDs advance only their one-way publication marker; replay, response loss, or restart leaves every unaccepted source eligible for deterministic re-poll.
- Hosted run `29906346469` isolated PostgreSQL's exact integer-literal/smallint function-resolution mismatch while proving the reward, ledger, session, and immutable source rows existed. Additive schema 0072 supplies only a strict immutable integer overload that delegates to the original event-ID function; the gameplay-preserving schema-71 trigger and history remain unchanged.
- Typed persistence repositories own session start/disconnect/reconnect/end and redacted client/server crash observations. Exact retries return stored facts, changed payloads fail closed, one open logical session is allowed per account, server-derived durations are canonical, and publication may advance only once.
- The persistent private-life server now owns an optional bounded logical-session coordinator. Accepted client platform, explicit environment/region configuration, `cohort.private`, durable-head restart inspection, generation-safe reconnect/clean-exit behavior, and graceful shutdown endings are captured without ticket access or any gameplay failure path. [`GB-M03-09-production-session-runtime-evidence.md`](../evidence/GB-M03-09-production-session-runtime-evidence.md) records the exact claim boundary.
- Protocol generation 1.24 and `telemetry.native-crash.v1` add a negotiated fixed-field native crash request/result. The production client retains one privacy-safe panic marker for next-launch delivery and deletes it only after durable acceptance; disabled, unavailable, conflict, or response-loss outcomes remain nonblocking and retain the marker. Server-owned predecessor-session recovery prevents a prior crash from being attributed to the healthy replacement session. [`GB-M03-09-native-crash-runtime-evidence.md`](../evidence/GB-M03-09-native-crash-runtime-evidence.md) records the exact redaction, authority, and verification boundary.
- The production `BoundCorePrivateLifeServer` owns a disabled `TelemetryWorker` from bind through shutdown. Disabled mode attaches no source or exporter, loads no pseudonymization secret, spawns no task, performs no poll/acknowledgement/export attempt, and contributes its explicit zero-residue report to the server's shutdown result. This keeps analytics outside gameplay availability while leaving the approved committed-adapter/exporter seam in one dedicated owner. [`GB-M03-09-disabled-production-worker-evidence.md`](../evidence/GB-M03-09-disabled-production-worker-evidence.md) records the exact boundary.
- When later approved and enabled, the worker exports privacy-safe documents through the existing idempotent exporter seam. Failed responses cause no source mutation. Exact accepted IDs advance existing `published_at` markers atomically; unaccepted rows survive process loss and are re-polled after restart.
- Additive migration `0073_m03_terminal_telemetry_origins_v1.sql` binds each newly committed death, extraction, Recall, and successor outbox row to exactly one durable session interval using PostgreSQL-authored transaction timestamps for both boundaries and terminal occurrence. Missing or overlapping coverage leaves the origin absent without rejecting gameplay; origin fields are immutable and historical rows are not guessed or backfilled.
- The terminal adapter now accepts only the separated pseudonymization key. It joins the captured session for all `TEL-001` context, derives death session duration from the immutable PostgreSQL start boundary, and derives item-power band from the committed death summary/destruction/item graph. Caller-supplied live session context has been removed.
- Compile-checked hosted journeys use the real death, extraction, Recall, and successor repositories and cover no-session fail-open, immutable origins, adapter loss and deterministic re-poll, exact acknowledgement, and the owning one-way publication guards. The focused origin test additionally covers application-clock skew, deliberately ambiguous database intervals, foreign-account absence, and half-open session handoff.
- `TEL-003` is not complete: the authoritative death transaction does not yet persist boss phase, party size/contribution, or detailed ping/jitter/loss/correction facts. These fields remain explicitly unavailable rather than receiving fabricated zero, solo, or healthy defaults; package closure waits for additive atomic death-source capture.
- [`ADR-039`](../decisions/ADR-039-telemetry-outbox-privacy-and-retention.md) records the privacy, ingestion, retention, and deletion boundary.

## Explicit exclusions

- Gameplay writes or decisions, direct live-handler emission, arbitrary JSON fields, raw crash dumps, raw identity lookup, or a second durable telemetry database.
- M04+ party/commerce/achievement events, dashboards, product KPI decisions, remote collector procurement, and legal/privacy approval.
