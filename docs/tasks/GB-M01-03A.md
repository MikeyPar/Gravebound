## Player-visible outcome

The Drowned Pilgrim enters through a readable spawn warning, approaches a present player until five tiles, locks aim at fan windup, fires the exact three-shot physical Chip fan, and repeats deterministically. Its simulation record, content references, attack facts, and reward binding are authoritative and renderer-independent.

## Exact rules

- Use the explicit FP override, not later production values: Fodder, health `85`, armor `0`, hurtbox radius `0.34`, speed `2.2`, aggro `10`, leash `12` tiles.
- State order is `SpawnTelegraph(900) -> Acquire -> ApproachUntilDistance(5) -> AttackWindup(300) -> FireFan -> Recover(1900) -> Acquire`.
- Aggro and five-tile stop boundaries are inclusive. While approaching, absent target or distance greater than the 12-tile leash returns to Acquire.
- Fan aim locks at windup start. Invalid zero aim retains the last valid aim rather than inventing a direction.
- Fan offsets are `-15/0/+15°`; origin offset `0.45`; speed `5.5`; radius `0.12`; lifetime `2.2 s`; physical Chip damage `8`; threat `3`; memory `fan_projectile`; counterplay `strafe`; disposition `consume_on_player_or_solid`; maximum active instances `6`.
- Spawn and hostile windup compile with ceiling-to-tick. Ordinary recovery/lifetime compile nearest. Exact resolved values are spawn `27`, windup `9`, recovery `57`, lifetime `66` ticks.
- Pattern and reward references each occur exactly once in the enemy and FP manifest. Authored milliseconds are validated before compilation, so same-rounded-tick drift fails.

## Ownership and failure behavior

- `sim_core::enemy` owns the immutable definition, stable cast IDs, state machine, aim lock, attack events, and fixed trace. It does not own rendering, health mutation, death, drops, or persistence.
- `sim_content` owns exact FP record/pattern/reward/manifest validation and compilation into the sim-core definition.
- Wrong ID, role, tuple, state order/duration, pattern kind/payload, damage metadata, cue/disposition/memory/threat, reward reference, manifest reference, duplicate record, invalid aim arithmetic, cast-ID overflow, or tick overflow fails closed.
- An `advance` failure must not partially commit state or emit a fabricated attack.

## Acceptance criteria

1. Strict content compiles the exact immutable FP definition and rejects semantic or same-tick authored drift.
2. The fixed state machine cannot attack before spawn/windup, locks one deterministic fan aim, and emits the exact attack payload/cadence.
3. The fixed trace is stable across processes and invalid inputs fail transactionally.
4. LocalLab presentation, hostile collision/damage, health/death/drop readability, optimized capture, and cumulative M01 gates pass.
