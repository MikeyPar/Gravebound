# GB-M03-04B — Production item and reward foundation

**Status:** In progress — strict schemas, semantic compilation, immutable development revision, and first production item math are complete; deterministic reward planning remains.

## Objective

Establish production-grade, typed item-template and reward contracts plus deterministic item math without authoring the Core 18-item catalog, allocating durable item identities, mutating inventory, or enabling a player route.

## Three design authorities

1. `Gravebound_Production_GDD_v1_Canonical.md`: `LOOT-001`–`005`, `LOOT-010`–`012`, `LOOT-020`, `LOOT-031`–`034`, and `TECH-100`–`102`.
2. `Gravebound_Content_Production_Spec_v1.md`: `CONT-ITEM-001`–`004`, `CONT-AFFIX-001`–`005`, `CONT-CATALOG-001`–`050`, `CONT-REWARD-001`–`005`, and `CONT-VALID-001`–`003`.
3. `Gravebound_Development_Roadmap_v1.md`: `GB-M03-04`, the M03 restart/item gate, and all seven owner-approved `SPEC-CONFLICT-007` resolutions.

## Completed slices

- Added strict production-style template, rarity-profile, typed reward-roll, material-pool, and Core stage-policy schemas alongside—never in place of—the immutable First Playable schemas.
- Template rarity and rolled affixes are absent by construction. Core behavior uses typed fixed-point Crossbow, Arbalist Relic, Armor, Charm, and consumable payloads rather than string stat names.
- The semantic compiler enforces exact target/reference closure, sorted unique identities/tags, class-slot-family pairing, level bounds, typed material/equipment separation, 100-point material pools, 10,000-basis-point rarity profiles, exact Core `85%/15%` usability, fixed Forged rarity, empty Core affix manifest, and disabled fragment/cosmetic checks.
- Compiled wipeable items expose only `core-dev.blake3.<full-manifest-hash>` as their immutable instance revision; no mutable development label or premature `core.1.0.0` exists.
- Pure integer item math implements all six rarity multipliers and exact Crossbow `W`, including level 1/10 boundaries and the canonical level-8 Grave Repeater result.

## Current implementation slice

Implement pure deterministic reward planning over compiled definitions with an injected bounded draw stream. Preserve authored draw order, choose level before legal template, keep equipment/universal/material categories disjoint, apply the explicit Core compatibility policy, and emit immutable plans without allocating item UIDs or mutating inventory.

## Deferred decision before durable rewards

The three authorities define draw order and the eventual RNG family but not the exact production reward seed preimage/domain separation. `GB-M03-04B` therefore uses injected deterministic draw tapes. Before `GB-M03-04D` commits server reward results, approve an exact seed contract; no implementation may infer one from the affix or dungeon seed formulas.

## Remaining acceptance work

- Injected-draw reward planner and deterministic plan hashing.
- Core candidate completeness across every authored source level and roll category.
- Full focused/workspace gates and 04B closeout audit.

## Exclusions

The exact 18 item records/assets/localization (`04C`); UID finalization, durable consumable units, placement, ledger, and ground expiry (`04D`); field equipment/UI (`04E`); CharacterSafe/Vault (`04F`); real-QUIC/PostgreSQL closure (`04G`); affix generation; route or Hall-station activation.
