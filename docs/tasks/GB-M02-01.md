# GB-M02-01 — Versioned protocol and QUIC handshake

## Objective

Create an implementation-ready, fail-closed network contract and prove that a real client and authoritative server can exchange it over TLS-authenticated QUIC before replicated gameplay is added.

1. `Gravebound_Production_GDD_v1_Canonical.md`: `TECH-010`–`TECH-015`, `SIM-004`, and `SIM-011`.
2. `Gravebound_Content_Production_Spec_v1.md`: `CONT-002`; bundle remains `fp.1.0.0`.

4. Repository `ADR-018-network-transport-codec-and-versioning.md`, which fulfills the roadmap decision without overwriting the existing M01 `ADR-003`.

## Implemented contract

- `ClientHello` carries protocol major/minor, bounded build ID, platform, 1–4 unique compression choices, exact lowercase 64-hex manifest hash, redacted 1–2,048-byte auth ticket, and bounded locale.
- `ServerHello` carries bounded session ID, negotiated version, required build, bundle, canonical 30/15 Hz rates, region, and at most 64 unique feature flags.
- Rejections are the exact nine GDD values. `HandshakePolicy` defines stable precedence and keeps ticket inspection outside policy/logging.
- Inputs carry nonzero sequence, tick, bounded fixed-point movement/aim, held primary, and monotonic action sequence fields. Recall/interaction use reliable `ActionFrame` messages.
- Snapshots are latest-state datagrams split into no more than 64 chunks of no more than 32 unique, valid entities.
- Reliable events cover action results, deterministic pattern descriptors with 64-bit seed, mutation results, control/time sync, and social pings. Each maps to exactly one `TECH-011` channel.
- Frames use the `GBN1` bounded postcard codec described by `ADR-018`. Header/payload kind and datagram semantics must match. A canonical frame BLAKE3 hash pins serialization bytes.
- `server_app::serve_handshake` and `bot_client::perform_handshake` exchange exactly one request/response on a reliable bidirectional QUIC stream.

## Acceptance

- Malformed lengths, magic, versions, transport flags, kind mismatches, invalid semantic values, duplicate IDs/options, and oversize data fail closed.
- Every required rejection is unit-tested, including a successful admission.
- A real Rustls-trusted QUIC loopback exchanges equal typed responses at client and server.
- Focused tests and all-target warnings-denied Clippy pass.
- Full workspace CI passes and README advances only after the audit records evidence.
