# GB-M01-06A - Local death transaction and fresh run

## Task contract

**Feature IDs:** `GB-M01-06A`, `COM-002`, `CONT-FP-001`, `CONT-FP-009`, `CONT-FP-010`
**Depends on:** `GB-M01-05A`; prototype inventory and run-entity ownership
**Decision:** [ADR-013](../decisions/ADR-013-local-death-freeze-and-fresh-run.md)
**Persistence:** none. This is an explicitly local, nonpersistent First Playable transaction.

## Required behavior

- Accept the exact authoritative lethal `DamageEvent`; never infer death or killer provenance from presentation.
- Resolve health zero before every later same-tick player action, accept death once, and freeze the old encounter.
- Retain a stable prior-ten-second trace containing source, pattern, raw/final damage, type, health before/after, source position, tick, and lethal state.
- Destroy all run-owned enemies, hostile projectiles/hazards, friendly projectiles, field pickups, reward entities, transient effects, equipped items, backpack stacks, and belt stacks.
- Reject duplicate death and restart while alive without mutation.
- Require explicit `Run Again`; rebuild from validated content rather than reset the dead runtime.
- Use the documented default seed and exact initial loadout: Pine Crossbow, Dented Scope, Reedcloth Wraps, empty Charm, and two Red Tonics.
- Qualify player, enemy, hostile-projectile, item, and death identities by run so no successor can alias the dead run.
- Return authoritative player control in less than three seconds and expose measured reconstruction time.

## Failure behavior

- Invalid pattern IDs, nonfinite source positions, lethal-flag disagreement, trace regression, cleanup overflow, identity overflow, tick mismatch, duplicate death, and illegal restart fail closed.
- A failed death transaction leaves lifecycle, inventory, encounter, and client runtime unchanged.
- A failed restart leaves the old run frozen; it never exposes a partially reconstructed successor.

## Acceptance criteria

1. Exact lethal ordering and later-action rejection remain covered by the 05A integration tests.
2. Death commits once, freezes the encounter, captures the lethal trace, and clears every owned inventory/entity category.
3. Explicit restart creates run 2 with fresh qualified IDs, full health, exact starter state, default seed, empty trace, and active control.
4. Invalid, duplicate, overflow, and deterministic replay fixtures are transactional and repeatable.
5. Strict workspace tests/lints/content validation, repeated deterministic traces, optimized build, warning-free runtime, inspected semantic evidence, and README handoff pass.

## Explicitly out of scope

- Durable `DTH-001` storage, account state, memorials, Fallen Hero Echo creation, item-destruction ledgers, networking, telemetry, full cause-of-death/summary UI, Escape/pause actions, audio, and blind-test execution.
