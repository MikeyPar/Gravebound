# GB-M03-02B — Durable account and character identity

**Status:** In implementation under approved `SPEC-CONFLICT-005` and accepted `ADR-029`.

## Objective

Replace the default Core identity server's process-local repository with the PostgreSQL adapter while preserving the exact `GB-M03-01` domain and protocol contract. A committed roster and selection must survive server restart; the explicit ephemeral adapter remains available only as a named regression mode.

## Three design authorities

1. `Gravebound_Production_GDD_v1_Canonical.md`: `UI-007`, `UI-008`, `TECH-004`, `TECH-006`, `TECH-020`, `TECH-021`, `TECH-030`, and `TECH-060`.
2. `Gravebound_Content_Production_Spec_v1.md`: immutable Grave Arbalist identity references, stable IDs, idempotent stored results, and no premature item/Core promotion.
3. `Gravebound_Development_Roadmap_v1.md`: `GB-M03-02`, `GB-M03-11`, restart/duplication exit gates, and the approved incremental ordering in `SPEC-CONFLICT-005`.

## Scope

- Asynchronous repository contract with both in-memory regression and PostgreSQL implementations.
- Private domain aggregate conversion to explicit typed storage rows; corrupt or incompatible rows fail closed.
- One serializable account transaction for bootstrap, create, select, optimistic version, and exact mutation-result replay.
- Default `serve-core-identity` durable mode using environment-only `GRAVEBOUND_DATABASE_URL`, embedded migrations, readiness validation, and zero combat admission.
- Explicit `serve-core-identity-ephemeral` regression mode that retains the `GB-M03-01` restart-wipe proof without masquerading as LocalStack.
- Real-QUIC PostgreSQL create/restart/bootstrap journey plus service-level replay/isolation coverage.
- Runnable identity-stage LocalStack with isolated Docker project or an explicit database URL.

## Exclusions

Hall/control transfer, item/vault rows, level/XP progression, oath selection, death/memorial/Echo state, currency ledgers, production identity, and formal Core promotion.

## Acceptance criteria

1. Identical create/select retries return their stored authoritative result; stale/conflicting/cross-account operations remain nonmutating under the same domain reducer used by the in-memory adapter.
2. A real QUIC server creates one legal character, stops, binds a new endpoint against the same database, and returns the exact roster with `persistence_enabled=true` and zero combat sessions.
3. A separate authenticated account remains isolated, corrupt rows fail closed, database credentials remain redacted/environment-only, and missing PostgreSQL fails with an actionable error.
4. Strict Clippy, focused/cumulative server tests, mandatory PostgreSQL CI, release build, docs, feature registry, and Current Next Step pass without adding later-domain schema.
