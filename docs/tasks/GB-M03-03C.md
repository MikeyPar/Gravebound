## Scope

- A renderer-independent `sim_core` world-scene module with validated northwest-origin bounds, radius-aware solid collision, safe volumes, roads, station/portal volumes, stable object IDs, and deterministic state hashing.
- A `sim_content` compiler adapter that converts only the immutable `core-dev-world-flow` records into simulation definitions; no duplicate hand-authored coordinates or runtime defaults.
- Exact Lantern Halls simulation: `64x48`, solid shell/desks/plinth, radius-0.30 player, default and character-select-return arrivals, five Core station projections, noncombat creation prohibitions, and direct navigability.
- Exact private microrealm simulation: `48x48`, solid shell, clear-mud floor, walkable Realm Gate, player spawn, width-five road, Lantern Fork safe circle, Bell portal area, and capacity one.
- A deterministic encounter-trigger seam for first movement beyond one tile or primary release, one-second delay, 900 ms warning request, active/reset/cleared lifecycle, five-second empty reset, and Bell-portal availability. `03D` supplies `pack.bell.01` entities and clear authority.
- Server-owned interaction projection with range 1.5, exact instant/500 ms semantics, one panel per player, authoritative disabled reasons, and no client-selected destination or enabled state.
- Native Bevy graybox scenes generated from the compiled definitions, with camera, local movement, collision, readable station/portal markers, safe-zone language, muted wet-stone/brass/ash palette, exact localized names, keyboard controls, and standard/reduced-motion behavior.
- Explicit disposable showcase/test entry only. Character Select `Play`, production Realm Gate admission, and `core_world_flow_integration` advertising remain disabled.
- Fixed-trace, geometry, collision, navigation, safety, interaction, lifecycle, content-drift, native render, 1920x1080, 1280x720, and reduced-motion evidence gates.

## Acceptance criteria

1. Runtime geometry is compiled from the exact `03A` records and rejects missing, extra, mismatched, out-of-bounds, overlapping, or semantically invalid scene data.
2. Hall movement cannot penetrate the shell, desks, or plinth; every enabled station remains reachable by the authored radius-0.30 player; hostile, damage, projectile, pickup, drop, and death creation are structurally unavailable.
3. Microrealm movement follows the exact shell and road geometry; Realm Gate, Lantern Fork, safe area, Bell portal, spawn, and capacity-one rules match the content specification without a macro scheduler, realm cycle, Siege, retirement, party, or public admission.
4. The trigger seam fires once with exact tick timing, survives deterministic replay, resets only after the specified empty interval, and exposes Bell portal availability only after an authoritative clear.
5. Interaction prompts/panels derive from authoritative range/state, use exact localization and typed disabled reasons, and support keyboard/pointer focus without allowing presentation to mutate simulation state directly.
6. Native standard, reduced-motion, and 1280x720 evidence is inspected at actual game scale for collision alignment, navigation, labels, safe/portal distinction, contrast hierarchy, and unobstructed playfield.
7. Focused and workspace tests, warnings-denied Clippy, strict content validation, deterministic traces, generated-schema checks, optimized Windows construction, and CI pass before the completion audit is recorded.
