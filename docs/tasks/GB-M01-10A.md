# GB-M01-10A - Accessibility effect controls

**Status:** PASS (local gate; GitHub intentionally excluded).
**Authorities:** GDD `UI-010`, `UI-030`, `ART-005/006/030`; content `COM-006/009`; roadmap `GB-M01-10A`.
**Scope decision:** use the exact M01 subset in the completion plan: shake, flash, reduced motion, high contrast, hostile theme, and friendly opacity.

## Required behavior

- Screen shake and localized flash accept `0..=100%` and default to `50%`; full-screen flashes are prohibited.
- Friendly-effect opacity accepts `10..=60%` and defaults to `35%` for the prototype.
- Reduced motion removes presentation smoothing without changing positions, timing, hitboxes, or simulation state.
- High-contrast telegraphs and three hostile themes retain structural fan/ring/lane shape cues; no mechanic depends on red/green distinction.
- Hostile telegraphs are never culled. Reduced-effects budgeting may remove only priority 5, then 4.
- `[F6]` opens a keyboard-operable overlay; `[1]..[6]` change one setting each. Settings remain presentation-only and outside gameplay hashes.

## Acceptance

Boundary/default/cycle tests, settings-safe simulation invariance, live real-hostile evidence, warnings-denied lint, optimized build, and the complete 1280x720/1920x1080 screenshot matrix must pass. See [the audit](../milestones/GB-M01-10A-audit.md).
