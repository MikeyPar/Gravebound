## Completed slices

- Strict independently hashed, non-promotable progression target with one exact level curve, one Grave Arbalist growth record, 13 XP profiles, six Core source bindings, and complete en-US reference closure.
- Exactly `xp.normal_t1`, `xp.miniboss_t1`, and `xp.boss_caldus` are Core-enabled. Realm Elite and every later-tier/event/boss profile remain authored but disabled.
- Pure deterministic simulation implements ordinary and `SOC-010` eligibility boundaries, integer contribution coefficients, Core XP saturation at 2,700, level derivation, first-clear rounding, Grave Arbalist growth, and nonhealing/nonlethal health rebuild.
- Migration 0003 adds normalized progression, immutable typed award evidence/results, account-wide boss-first-clear ownership, and progression entry-restore children.
- Existing wipeable characters backfill only from the approved level-1 state. New character creation seeds progression atomically, and identity reads reject a divergent temporary level cache.
- The replay-first PostgreSQL award transaction and server service atomically bind typed evidence, exact outcomes, progression/cache updates, and account-wide first-clear markers. Replays return stored results before current-state checks.
- Protocol 1.8 exposes only server-authored progression snapshots/events. The persistent Core endpoint serves owned, living-character projections over reliable QUIC and fails closed on invalid content, ownership, life, security, or storage state.
- The native selected-character surface requests the exact progression content revision and renders current/max health plus level/XP using numbers, segmented shape, explicit low/critical labels, and an unambiguous Core-cap state.

## Completed restore slice

- Owner-approved restore semantics bind every danger XP result to the exact active entry restore point while safe-location results remain unbound.
- Schema 13 preserves immutable award evidence, adds exact revocation metadata and an idempotent restore completion version, and rejects null/unknown revocation shapes.
- Fresh awards lock and derive the authoritative world-location binding inside the same serializable transaction; callers cannot provide a restore identity.
- The real progression provider captures exact entry level/XP/health/version, restores those values with a monotonic live version, revokes only exactly bound receipts, and deletes only first-clear markers owned by those receipts.
- Replays of revoked receipts return typed `revoked_by_crash_restore` with no stale projection; conflicting canonical material remains an idempotency conflict.
- Living/active restore-root/location checks make committed death or extraction win without progression mutation.
- The disposable PostgreSQL fixture is compiled and warnings-clean but cannot execute in the current environment because its required database URL and destructive-test opt-in are absent.
