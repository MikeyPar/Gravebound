# GB-M01-01A — Bevy camera and world units

## Task contract

**Task ID:** `GB-M01-01A`  
**Title:** Bevy camera and world units  
**Milestone:** M01 — First Playable  
**Feature IDs:** `GB-M01-01A`

**Player-visible outcome:** Opening LocalLab shows the authored Bell Laboratory at a stable top-down scale, centered on the future player spawn, with readable debug geometry for the shell, pillars, player/boss spawns, reward points, and wave anchors.

**In scope:**

- Immutable simulation-owned arena geometry compiled from `arena.prototype.bell_laboratory_01`.
- Northwest-authored to centered-render coordinate conversion.
- One authored tile equals one simulation and render-world unit.
- Orthographic camera with `13.5` visible tiles vertically (`24` horizontally at 16:9).
- Exact `32 × 24` walkable interior, one-tile shell, four pillars, spawns, and debug anchors from `CONT-FP-002`.
- Runtime content validation, debug grid, labels, tests, screenshot review, and release smoke test.

**Out of scope:**

- Input actions, movement, camera follow/smoothing, zoom controls, collision response, enemies, combat, audio, or persistence.
- Production art assets; this task uses intentional debug primitives and the approved concept palette.

**Authoritative owner:** `sim_core::arena`, populated only by validated `sim_content`.  
**Client presentation owner:** `client_bevy::arena_view`.

**Dependencies:**

- Code modules: `sim_core`, `content_schema`, `sim_content`, `client_bevy`.
- Content IDs/schemas: `arena.prototype.bell_laboratory_01`, Arena schema v1, `fp.1.0.0` manifest.
- Previous tasks: `GB-M00-01`, `GB-M00-06`, `GB-M00-07`.

**Exact rules and prototype defaults:**

- `SIM-001`: one tile is `1.0` gameplay unit; authored origin is top-left; calculations never use pixels.
- `SIM-002`: top-down orthographic camera; default visible area approximately `24 × 13.5` at 16:9.
- `CONT-010`: `+x` east and `+y` south in authored coordinates.
- `CONT-FP-002`: exact bounds, shell, pillars, points, and anchors are data, not client constants.
- Render conversion is `render_x = authored_x - width/2`, `render_y = height/2 - authored_y`; only the client owns this presentation transform.

**Legal state transitions:** `Boot -> ValidateContent -> BuildArenaView -> Running`; window close transitions `Running -> Stopped`.

**Rejected inputs and typed errors:** zero dimensions/shell, invalid rectangles, out-of-bounds points, duplicate/empty anchor IDs, overlapping pillars, missing arena ID, and any mismatch from the exact First Playable geometry fail before the Bevy app starts.

**Network messages/version behavior:** none; LocalLab is single-process and nonpersistent.

**Persistence and idempotency:** none.

**UI states:**

- Loading: process startup before content validation; no interactive frame is exposed.
- Empty: illegal; the exact arena must exist.
- Success: rendered arena with diagnostic header and legend.
- Disabled: debug labels may later be placed behind `GB-M01-08A`; always enabled for this task.
- Error: typed startup error on stderr and nonzero process exit.
- Disconnected/Retry: not applicable before M02; restart process after correcting content.

**Telemetry events/fields:** no product telemetry before `GB-M01-10B`; startup logs content version, content hash, arena ID, and geometry counts.

**Accessibility/readability:** no hostile red is used for non-hostile grid geometry; shell, pillars, player spawn, boss spawn, wave anchors, and utility anchors have distinct luminance and shapes; labels supplement color.

**Tests required:**

- Unit: geometry bounds, shell expansion, duplicate/overlap rejection, coordinate conversion, fixed camera scale.
- Deterministic: authored-to-render results are independent of record order.
- Integration: checked-in `fp.1.0.0` compiles to the exact geometry.
- Adversarial/failure: malformed geometry and missing arena fail validation.
- Content validation: existing strict package validation plus exact arena assertions.
- Visual/manual: launch release client, inspect a captured 16:9 frame, verify process responsiveness.

**Feature flag/kill switch:** none; this is the only M01 runtime surface.  
**Rollback/migration behavior:** revert the feature commit. Changing geometry requires a content-spec update, bundle migration, and updated fixtures.

## Acceptance criteria

1. The checked-in `32 × 24` northwest-origin arena compiles into one authoritative geometry object with exact shell, pillars, spawns, and anchors.
2. At 16:9, the orthographic view is exactly `24 × 13.5` render units and preserves one tile as one render-world unit at other aspect ratios.
3. Unit, integration, content, visual, clean CI, and Windows release smoke checks pass without duplicating authored coordinates in client code.

## Files/modules allowed to change

- `sim_core`, `sim_content`, `client_bevy`, content validation tests, development/task documentation, README, and CI only if verification requires it.

## Completion report requirements

- Changed files, tests and results, acceptance evidence, screenshot/runtime evidence, and any remaining edge case or specification conflict.
