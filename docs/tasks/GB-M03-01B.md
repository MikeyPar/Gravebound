# GB-M03-01B — ephemeral Core account and character authority

## Status

Implemented; automated authority and real-QUIC gates pass. Parent package `GB-M03-01` remains open for native presentation and combined evidence.

## Contract

Implement the server-authoritative, explicitly wipeable account/character slice required by the canonical GDD, Content Production Specification, Development Roadmap, approved `SPEC-CONFLICT-004`, and parent `GB-M03-01` contract.

## Delivered

- Protocol 1.6 appends bounded Control bootstrap and Mutation create/select messages while preserving existing discriminants and canonical M02 compatibility bytes.
- Safe snapshots expose version, capacity, roster, selected character, and deterministic roster ordinal. They contain no account ID, name, appearance entitlement, item power, last-played time, oath choice, or Play transfer.
- Authenticated ticket hashes bind opaque server-side test accounts. Production namespace, unauthenticated, content-mismatched, forged-character, stale, conflicting, malformed-hash, and future-issued requests fail closed with typed results.
- The reusable repository transaction seam protects aggregate version, roster, selection, and a bounded non-evicting idempotency ledger under one writer lock.
- Creation permits only `class.grave_arbalist` and produces exact level-1/no-oath/living/safe defaults in one of two slots. Selection permits only a living owned character and does not grant world control.
- The explicit Core identity runtime advertises only the unpromoted `core-dev` target and never constructs M02 combat authority.
- Bot helpers exercise the same QUIC framing used by native clients.

## Explicitly deferred

PostgreSQL/durable restart survival (`GB-M03-02`), Hall/world transfer (`GB-M03-03`), items/item power (`GB-M03-04`), oath selection (`GB-M03-05`), death/memorial/successor flows, editable names, appearance entitlements, and formal `core.1.0.0` promotion.

## Acceptance

- Protocol and domain tests prove bounds, exact defaults, two slots, deterministic injected IDs/clock, accepted duplicate retry, conflicting retry, stale version snapshot, invalid class, payload hash mismatch, issue-time validation, account isolation, cross-account rejection, and privacy redaction.
- Real QUIC proves handshake, bootstrap, create, same-process reconnect recovery, zero combat admission, and a clean-server restart wipe.
- Strict formatting and Clippy pass for the owned protocol/server/bot production libraries. Full parent-package lint and visual gates remain owned by `GB-M03-01C` and the combined audit.
