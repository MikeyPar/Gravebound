# GB-M01-07A - Deterministic prototype inventory and reward actions

## Task contract

**Feature IDs:** `GB-M01-07A`, `LOOT-001`, constrained `LOOT-002`, `CONT-FP-010`
**Depends on:** M00 item schema; `GB-M01-02A`; `GB-M01-06A` run ownership
**Persistence:** local run only.

## Required behavior

- Own exactly Weapon, Relic, Armor, Charm, eight nonbelt backpack stacks, and two belt slots.
- Field Take places into the lowest empty backpack index. Explicit Equip uses the authored slot and moves the replaced item to the lowest empty backpack index.
- Automatic field collection requires distance `<=0.75` tiles; explicit Interact requires distance `<=1.25` tiles. Equality is eligible and out-of-reach attempts are nonmutating.
- Reject an Equip swap when the backpack is full; never destroy either item.
- Keep a capacity-blocked field pickup available for the remainder of its exact 60-second lifetime.
- Reward Take/Equip use the identical placement rules. Reward actions additionally support Drop Existing, Leave Reward, and explicit capacity rejection.
- Tonic pickup merges belt slots in ascending order, then a matching nonfull backpack stack, then the lowest empty backpack index.
- Every operation is clone-before-commit, rejects duplicate item instances, and is deterministic under replay.
- The inventory overlay states behavior before flat values, is keyboard toggleable, and protects the center/lower-middle playfield.
- Death/restart destroys every added stack through the 06A transaction.

## Acceptance criteria

1. Slot/capacity, Take, Equip, swap, full rejection, expiry, duplicate, Tonic, reward parity, Drop Existing, Leave, and deterministic replay tests pass.
2. A real canonical prototype field pickup enters the authoritative inventory through the client boundary.
3. A keyboard-operable reward panel proves Take, Equip, Drop Existing, Leave, and blocked outcomes without silent destruction.
4. Restart cleanup proves no field/reward stack survives.
5. Cumulative local gates and inspected optimized field/reward evidence pass.
