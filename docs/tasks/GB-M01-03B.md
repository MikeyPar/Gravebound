## Player-visible outcome

The stationary Bell Reed enters through a spawn warning and dormant beat, then emits deterministic six-projectile rings with a rotating adjacent two-index gap. First and repeated warnings use their exact distinct durations.

## Exact rules

- FP override: Pressure, health `130`, armor `2`, hurtbox `0.42`, speed `0`, aggro `11`, leash `12` tiles.
- State order: `SpawnTelegraph(900) -> Dormant(500) -> RingTelegraph -> FireGapRing -> RecoverUntilCycle(3000)`.
- The first telegraph is `450 ms`; repeated telegraphs are `300 ms`. The three-second cycle is anchored at telegraph start, not fire/recovery completion.
- Define eight indices at `45°`; omit adjacent indices starting `[0,1]`; advance omitted start `+3 mod 8` per cast; emit exactly six.
- Projectiles: speed `4.5`, radius `0.13`, lifetime `3 s`, veil Chip damage `10`, threat `6`, memory `radial_projectile`, counterplay `follow_gap`, disposition `consume_on_player_or_solid`, maximum active instances `12`.
- Resolved values: spawn `27`, dormant `15`, cycle `90`, first warning `14` (ceiling), repeat warning `9`, lifetime `90` ticks.
- Pattern/reward/manifest references are unique and exact. Same-rounded authored drift, wrong optional lane fields, cue IDs, kind, band/type, or gap payload fails closed.

## Ownership and failure behavior

- `sim_core::enemy` owns definition, cast IDs, omitted-index progression, telegraph/fire/recover events, and deterministic trace.
- `sim_content` validates the exact FP override and pattern before constructing the sim definition.
- No renderer, player collision/damage, health, death, reward roll, pickup, or persistence is owned here.
- Tick/cast overflow and invalid definition are typed failures; state is not partially committed.
