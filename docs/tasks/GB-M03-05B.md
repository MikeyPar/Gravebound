# GB-M03-05B — Durable initial Oath selection

**Status:** Complete — implementation, authorized disposable-PostgreSQL execution, real-QUIC restart evidence, and the completion audit pass.

## Objective

Deliver the level-10 Grave Arbalist initial-Oath choice as a server-authoritative, durable, explicitly confirmed character-life mutation without enabling later paid changes, the normal player route, or unapproved Oath content.

## Three design authorities

1. `Gravebound_Production_GDD_v1_Canonical.md`: `SIM-003`, `CLS-020`, the level-10 Oath rule, Hall-only changes, mutation exclusion, and the 40-Ash later-change boundary.
2. `Gravebound_Content_Production_Spec_v1.md`: the Core Oath allowlist, `CONT-HUB-002`, immutable content and localization rules, and deterministic validation requirements.
3. `Gravebound_Development_Roadmap_v1.md`: `GB-M03-05`, the M03 durability/retry exit gates, and the requirement to present both exact Grave Arbalist choices.

## Completed slices

- Protocol 1.9 defines bounded control-channel eligibility views and mutation-channel initial-selection frames/results. The canonical payload binds character, exact Oath, three-part content revision, explicit confirmation, expected character version, mutation identity, issuance time, and payload hash.
- Migration 0006 restricts Core lives to Long Vigil or Nailkeeper, records immutable bounded mutation results, and writes one durable `oath_selected` character-life outbox event for an accepted first choice.
- The replay-first serializable repository locks account, character, authoritative location, inventory root, and live inventory rows in stable order. It derives inventory safety inside the same transaction and cannot accept a caller-authored safety claim.
- Eligibility requires the authenticated account's selected living character, level 10, an exact Lantern Halls location, matching character/location version, the compiled content revision, no unresolved security state, and safe inventory. Missing or malformed inventory aggregates fail closed.
- Accepted selection advances one character-life version across character and location state. Identical mutation retries return the stored result; changed canonical material under the same mutation identity returns a typed idempotency conflict.
- Later changes remain explicitly stage-disabled. The first selection is free and no Ash wallet, paid-change path, or second-change behavior is implied.
- Persistent Core runtime construction loads the exact compiled catalog and routes both reliable messages to the PostgreSQL authority. Process-local Core recognizes the protocol but returns typed service-unavailable results.
- The real-QUIC disposable fixture selects Long Vigil, proves identical replay, restarts the server, and proves the selected projection persists. Bot helpers exercise the production reliable route rather than calling services directly.
- The native Core identity surface renders both exact localized choices, requires a separate review state with the permanent-life warning, and emits no mutation until mouse or keyboard confirmation. Cancel and every typed rejection state say that no permanent choice was committed.
- Full `cargo test --workspace --all-targets` and workspace-wide `cargo clippy --workspace --all-targets -- -D warnings` pass after the confirmation UI integration. The gate includes 72 native client tests and compiles the ignored disposable-PostgreSQL fixtures.
