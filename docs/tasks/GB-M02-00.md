# GB-M02-00 — Network workspace and ownership foundation

## Objective

Add the exact M02 workspace crates required by the canonical architecture and make each boundary runnable, testable, fail-closed, and honest about deferred transport work.

## Authorities

1. `Gravebound_Production_GDD_v1_Canonical.md`: `TECH-001` through `TECH-006`, `TECH-010` through `TECH-015`, and `TECH-060`.
2. `Gravebound_Content_Production_Spec_v1.md`: `CONT-002`; M02 introduces no new content bundle and continues to pin `fp.1.0.0` until M03 promotes `core.1.0.0`.
3. `Gravebound_Development_Roadmap_v1.md`: `GB-M02` objective/exit gate and package `GB-M02-00`.

## Scope

- Add workspace crates `protocol`, `server_app`, and `bot_client`.
- Pin Tokio as the server/bot async runtime.
- `protocol` owns protocol versions, wire-facing primitives, channel reliability, rates, and typed protocol errors; it owns no sockets or gameplay.
- `server_app` owns sessions/instances/routing and authoritative invocation of `sim_core`; it owns no rendering, client settings, gameplay rules, or database implementation.
- `bot_client` owns headless input/journey behavior over the real protocol; it cannot author outcomes or bypass authority.
- Pin exact GDD channel/reliability mappings, 30/15/20 Hz rates, 100 ms interpolation, 5-second sync, and 8-tile/4-tile interest constants.
- Provide server/bot doctor commands that validate their boundaries and explicitly report future transport/database/journey features as disabled.
- Add focused CI command coverage and preserve the full workspace gate.
