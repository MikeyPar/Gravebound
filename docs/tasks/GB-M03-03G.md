## Objective

Compose the completed M03 authorities into one persistent, ordinary, wipeable Core runtime and one native application route. This package is complete only when a private tester can play Character Select -> Hall -> microrealm -> fixed Bell dungeon -> Sir Caldus -> extraction/death -> Hall/successor without developer commands and with every mutation remaining server-authoritative and durable.

## Implementation slices

### 1. Normal world-flow authority

- Route location queries through the durable read projection.
- Route `CharacterSelect -> Hall`, Hall return-to-select, Hall Realm Gate -> `world.core_microrealm_01`, and cleared Bell portal -> `dungeon.bell_sepulcher` through authenticated, versioned, replay-first mutations.
- Preserve one lineage/restore root across microrealm-to-dungeon transfer. Capture a restore point only on safe-to-danger entry.
- Reject uncleared portal use, invalid source/range/content, later destinations, BB1/BS1, and seeded/public allocation without mutation.

### 2. Persistent normal server composition

- Add a dedicated `BoundCorePrivateLifeServer`; retain `BoundCoreIdentityServer` as the fail-closed regression endpoint.
- Construct persistent identity, progression, death view, Oath/Bargain, safe inventory, ResolutionHold, successor, world-flow, Recall, terminal, and scene authorities before advertising their capabilities.
- Own transport/session generations, reliable response sequencing, terminal completion delivery, actor cleanup, and zero-residue shutdown.

### 3. Append-only route projection and live private-life actor

- Append a negotiated `CorePrivateRouteStateV1` reliable event after all existing discriminants. It exposes only server-owned lineage, scene/room identity, phase, readiness, and state version.
- Own exact compiled Hall and microrealm movement/interactions, capacity-one admission, `pack.bell.01`, Bell portal clear state, the fixed `B0 -> B6` room graph, Core enemies/minibosses, Bargain rest room, Sir Caldus, rewards, and pending inventory.
- Feed authoritative snapshots/events through existing bounded protocol channels. Client input never authors collision, targets, damage, rewards, room completion, destinations, or terminal outcomes.
- Integrate the shared five-producer terminal coordinator, durable death, extraction, explicit/LinkLost Recall, crash restore, and process-resume behavior.

### 4. Ordinary native route

- Add one `core-private-life` client mode using the real negotiated server, Core identity roster, `CoreWorldTransitionModel`, compiled scenes, combat HUD, inventory/Vault, Oath/Bargain, death/Memorial, successor, Recall, and ResolutionHold projections.
- Add a real Character Select `Play` action and Hall Realm Gate interaction only when exact capabilities and authoritative state permit them.
- Require matching location and `CoreSceneReadiness` before control; preserve protected playfield, accessibility, keyboard/pointer focus, and standard/reduced-effects parity.
- Keep all standalone showcase commands isolated from the normal state machine.

### 5. Evidence and closure

- Unit, property/fixed-trace, transactional PostgreSQL, real-QUIC, reconnect/restart, malformed/adverse, performance, cleanup, and visual tests.
- Twenty-five ordinary scripted full loops without fixture-only route commands, covering extraction, Recall, lethal death, Memorial, successor, and return to checked danger control.
- Optimized native captures at 1280x720 and 1920x1080 in standard and reduced effects, with hashed build/content/evidence manifests and updated README screenshots.
- Updated tester package, parent `GB-M03-03` audit, and hosted cumulative CI before removing the normal-route gate.

## Slice 1 implementation record

- `CorePrivateWorldFlowRouter` preserves distinct durable location-read and transactional mutation owners without altering authenticated frames.
- `Blake3WorldFlowIds` derives restart-stable, domain-separated transfer, lineage, and restore identities from authenticated account, selected character, and mutation identity. Completed `03B`/`03F` fixtures retain a fail-closed compatibility alias.
- The exact Hall Realm Gate performs the existing safe-inventory preflight and captures one complete danger root. The Bell transfer preserves that lineage and restore point, does not move safe aggregate versions, and targets only `dungeon.bell_sepulcher`.
- Bell admission uses a two-phase actor reservation: validate and explicitly release PostgreSQL locks, prepare an exact content/lineage/character binding with nonzero permit, actor generation, and route-state version, and retain a non-clone owned lease that pins that generation. The prepare contract requires an internal RAII guard before any suspension so cancellation cannot strand a reservation. A fresh serializable transaction then revalidates and commits before bounded commit/abort handling consumes the lease after lock release. Committed death, actor replacement, or other durable drift therefore cannot pass between phases; timeout/cancellation drops the internal guard or returned lease; response-loss replay uses bounded reconciliation without acquiring a new permit.
- Recognized production portals from the wrong scene return `InvalidSource`; uncleared/range denials are durably replayable; transient actor failures remain retryable; all later destinations remain `DestinationDisabled`.
- Exact source `26f9c66` passes hosted CI [`29616169514`](https://github.com/MikeyPar/Gravebound/actions/runs/29616169514): Linux formatting/strict lint/tests/content/schema validation, the mandatory PostgreSQL suite, optimized Windows construction, and optimized native evidence are green. The PostgreSQL proof covers restart replay, changed-binding conflict, same-root preservation, all five V3 restore components, unchanged safe versions, same-aggregate actor reads without lock inversion, an exclusive concurrent permit whose loser receives `TransferInProgress` and then replays the one receipt, cancellation during prepare followed by generation release and successful exact retry, stable/transient actor decisions, and actor bypass for invalid/stale/content-mismatched requests.

## Slice 3 actor/content/generation record

- `CorePrivateLifeContent` loads and validates only the compiled Core Hall, capacity-one microrealm, fixed encounter-room set, `layout.core_private_life_01`, and Sir Caldus authority. Its length-prefixed, domain-separated aggregate revision covers the ordered world-flow, room, boss-record, asset, and localization inputs; disabled BB1/BS1 branches and any identity/capacity drift fail closed without promotion or route admission.
- `CorePrivateRouteActor` is the capacity-one route-phase authority for one authenticated account and selected character. It owns the exact Hall -> microrealm -> B0-B6 -> Caldus -> terminal phase graph, room safety/readiness phases, monotonic state versions, bounded mailbox, RAII Bell permit, cancellation cleanup, reconciliation without rewind, one active generation per account, and zero-residue shutdown. Commit `b02f0f6` routes ordinary advances and Bell prepare/commit/reconciliation through that mailbox; synchronous lease drop only releases the reservation token and cannot mutate route state. Extraction, Recall, or death retires the danger generation; returning to Hall requires a newly allocated higher generation rather than mutating the terminal actor backward.
- This phase actor does not yet own compiled movement, live combat, reward grants, pending inventory, or terminal execution. Those live owners remain explicit slice-3 work and are prerequisites for truthful normal capability advertisement.
- PostgreSQL migration `0062_private_route_actor_generations_v1.sql` and `PostgresPersistence::allocate_private_route_generation_v1` allocate strictly increasing generations for the authenticated selected living character under serializable retry. The immutable allocation audit and deferred head-closure constraint prevent reuse, foreign/dead-character allocation, audit mutation, or a committed head without its matching audit; gaps after ambiguous response loss are legal and never reused.
