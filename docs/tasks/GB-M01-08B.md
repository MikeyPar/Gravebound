# GB-M01-08B — Developer time scale and invulnerability

## Task contract

**Feature IDs:** `GB-M01-08B`, Roadmap package 08, `PRD-123`, `TECH-001`, `SIM-010`, `SIM-011`
**Depends on:** `GB-M01-08A` authoritative runtime/debug surfaces
**Persistence:** developer-only; disabled by default; excluded from gameplay hashes and ineligible gate sessions.

## Required behavior

- F4 cycles paused/single-step-capable, `0.25x`, `0.5x`, `1x`, and `2x` using the exact order in ADR-016; F8 restores `1x`.
- F6 advances one and only one authoritative 30 Hz tick while paused.
- Time scaling changes accumulation rate only; it never changes fixed-tick duration or deterministic system order.
- F5 enables invulnerability after collision and hypothetical damage calculation.
- Invulnerability rejects health, barrier, Focused/status, restore, and combat-state mutation while preserving contact, damage, projectile disposition, and grace events marked `debug_invulnerable`.
- Controls are enabled only by an explicit developer environment flag or screenshot-only fixture and start disabled/`1x`/vulnerable in ordinary play.
- Any non-`1x`, invulnerability, or single-step use excludes the session from gate metrics.
- The overlay clearly shows `DEBUG ONLY`, bindings, scale, invulnerability, step count, whole-tick behavior, and metric eligibility.

## Acceptance criteria

1. Default and cycle/state tests pass exactly.
2. Standard callers retain the unchanged damage policy by default.
3. Direct, projectile, and lane integration fixtures preserve real contacts/hypothetical damage and apply zero health/status mutation under debug invulnerability.
4. Projectile consumption and stable event order remain intact.
5. Optimized evidence visibly shows active `0.5x`, invulnerability, whole-tick labeling, and metric exclusion.
6. The cumulative local quality gate passes.

## Current implementation state

Completed per ADR-016. The engineering/evidence package and its full `GB-M01-08A` dependency pass locally; developer-altered sessions remain excluded from gate metrics.
